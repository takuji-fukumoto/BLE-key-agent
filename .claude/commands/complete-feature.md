# 機能実装完了

## 手順

以下の手順を順番に実行してください。

### Step 1: 実装状況の確認

- TodoWriteで全てのTODOを `completed` に更新してください
- 未完了のタスクがある場合はユーザーに報告し、続行するか確認してください

### Step 2: ドメイン知識の抽出・保存

実装した内容を振り返り、以下に該当する重要事項を `docs/spec/` 配下にfeature別のmdファイルとして保存してください:

- BLE通信やGATTに関する実装上の制約・注意点
- ライブラリ（bleak, bless, pynput, flet等）の挙動で発見したこと
- プロトコルやデータフォーマットの設計判断とその理由
- async/awaitやスレッド間通信のパターン
- ハードウェア依存の注意点
- その他、将来の開発者が知るべきドメイン知識

ファイル命名規則: `docs/spec/<feature名>.md`

フォーマット:
```markdown
# <機能名>

## 概要
<この機能が何をするか>

## 重要事項

### <トピック1>
<詳細>

### <トピック2>
<詳細>

## 関連ファイル
- `<変更したファイルパス>`
```

特に保存すべき重要事項がない場合は、このステップはスキップしてください。

### Step 3: コミット・プッシュ

- 変更内容を確認し、適切な粒度でコミットしてください
- docs/spec/ のドメイン知識ファイルも含めてコミットしてください
- リモートにプッシュしてください

### Step 4: PR更新

現在のブランチに紐づくDraft PRの本文を更新してください:

- `## 変更予定` を `## 変更内容` に書き換え、実際の変更内容を記載
- チェックリストの該当項目にチェックを入れる
- 必要に応じてDraftを解除する（`gh pr ready`）

`gh pr edit` を使ってPR本文を更新してください。

### Step 5: 完了報告

以下をユーザーに報告してください:
- コミット内容の要約
- PRのURL
- 作成したドメイン知識ファイル（あれば）
