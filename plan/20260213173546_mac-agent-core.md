# Phase 4: Mac側コア（BLE + キー監視）実装

## 概要

Mac側のコア機能を実装します：
- **KeyMonitor** (`src/mac_agent/key_monitor.py`): pynputによるグローバルキー入力監視とasyncio連携
- **BleClient** (`src/mac_agent/ble_client.py`): bleakによるBLE Central管理（スキャン、接続、送信、再接続）

この実装により、キー入力がBLE経由でRaspberry Piに到達します（CUI確認可能）。

## 実装の重要ポイント

### 1. pynput→asyncio Bridge（KeyMonitor）
```python
# pynputコールバック（別スレッド）からasyncio event loopへの安全な橋渡し
def _on_press(self, key):
    event = self._create_event(key, is_press=True)
    if self._loop and self._queue:
        asyncio.run_coroutine_threadsafe(
            self._queue.put(event),
            self._loop
        )
```

### 2. Lazy Import（BleClient）
```python
# 型ヒント用（実行時インポートなし）
from typing import TYPE_CHECKING
if TYPE_CHECKING:
    from bleak import BleakClient

# 実行時インポート（メソッド内）
async def connect(self, address: str) -> bool:
    from bleak import BleakClient  # Pi環境でのエラー回避
```

### 3. Write Without Response（低レイテンシ）
```python
await client.write_gatt_char(
    KEY_CHAR_UUID,
    data,
    response=False,  # ACK不要、低レイテンシ優先
)
```

### 4. 指数バックオフ再接続
```python
delay = 1.0
while True:
    await asyncio.sleep(delay)
    if await self.connect(self._last_address):
        return  # 成功
    delay = min(delay * 2.0, 60.0)  # 1→2→4→8→...→60秒
```

### 5. コールバック例外分離
```python
def _set_status(self, new_status: str):
    self._status = new_status
    if self._on_status_change:
        try:
            self._on_status_change(new_status)
        except Exception:
            logger.exception("Error in callback")  # クラッシュしない
```
