- 作成日: 2026-03-03
- ブランチ: `feature/unihiker-m10-sample`
- 目的: 既存 `sample/raspi_receiver/apps/lcd_display` をベースに、Unihiker M10 の GUI ライブラリを使った受信表示サンプルアプリを追加する。
- 方針: `raspi_receiver/lib` は変更最小限とし、Unihiker 固有実装は `sample/unihiker_receiver/` に閉じ込める。

### 1) 導入・初期化

- ライブラリ名: `unihiker`（UNIHIKER本体にはデフォルト導入済み、必要に応じて `pip install -U unihiker`）
- GUI利用の基本形:
  - `from unihiker import GUI`
  - `gui = GUI()`

### 2) 画面表示API（今回使う候補）

- `GUI.draw_text(...)` でテキストオブジェクトを生成
  - 主な引数: `x`, `y`, `w`, `text`, `color`, `origin`, `onclick`, `font_family`, `font_size`, `angle`
  - 戻り値: Text object
- 生成したオブジェクトは `object.config(...)` で更新可能
  - 例: `label.config(text="...")`
- 不要オブジェクトは `GUI.remove(object)` または `object.remove()` で削除
- 全消去は `GUI.clear()`

### 3) 入力イベントAPI（今回使う候補）

- `gui.on_a_click(callback)` / `gui.on_b_click(callback)`
- `gui.on_key_click(key_name, callback)` でキーイベント登録（例: `<space>`）
- `wait_a_click()` / `wait_b_click()` の待機系APIも存在

### 4) 実装上の注意点（公式記載ベース）

- 画面座標は 240x320（originは左上、x右方向/y下方向）
- 原点指定 `origin`（`top`, `bottom`, `left`, `right`, `center`, `top_left` など）を活用してレイアウト固定
- `add_button` の `onclick` 内で GUI が即時更新されない場合があり、必要に応じて `start_thread(...)` 利用を検討
- 本サンプルは BLE受信 (`KeyReceiver`) を主経路とするため、Unihiker側のキー入力APIは補助（デバッグ/終了操作）用途に限定

### 5) この計画への反映

- Phase2 で `draw_text + object.config` を前提に GUIアダプタを設計
- Phase3 で BLE受信イベントを GUI更新にマッピング
- Phase5 で `sample/scripts/setup_unihiker_sample.sh` に `unihiker` ライブラリ確認/更新手順を含める
